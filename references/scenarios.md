# 老板指令模式参考库

> 帮助 AI 理解"当老板下指令"的常见表达模式。
> 按指令风格分类。

---

## 一、代词式指令（用"它/那/这个/那个"代替具体对象）

**核心特征：** 老板默认你知道他在说什么。

| 模式 | 例子 | AI 解读策略 |
|------|------|------------|
| 它 + 动作 | "修好它"、"跑一下它"、"查查它什么情况" | 最近讨论的对象 |
| 那/那个 + 名词 | "那个 API 好了没"、"那个方案我看一下" | 最近提到的对应名词 |
| 这/这个 | "这个改一下"、"你看看这个" | 当前话题或最近的文件 |
| 那东西/那玩意儿 | "那东西能用吗"、"那玩意儿谁写的" | 上轮对话的核心对象 |
| 这几个 | "这几个一起改了吧" | 上文列举的多个对象 |

**校准变量：** `pronoun_inference`
- 0.5-0.7：偶有代词错误，部分确认
- 0.8-1.0：代词几乎完全准确，直接推断

---

## 二、裸命令式指令（只有动词，没有宾语）

**核心特征：** 老板不说"做什么"，默认你知道。

| 模式 | 例子 | AI 解读策略 |
|------|------|------------|
| 单动词优化 | "优化"、"改进"、"重构" | 最近提及的代码/流程 |
| 动词+程度 | "优化一下"、"稍微改改"、"大改" | 同上 + 改动力度 |
| 检查类 | "查一下"、"看看"、"检查"、"验证" | 最近出问题的东西 |
| 部署类 | "发版"、"上线"、"发布"、"deploy" | 当前分支/项目 |
| 状态类 | "怎么样了"、"进度"、"什么情况"、"status" | 正在进行的工作 |
| 极端简短 | "好"、"继续"、"下一步"、"go" | 继续上一步操作 |

**校准变量：** `bare_command_tolerance`
- 0.3-0.5：裸命令不可接受，必须确认
- 0.6-0.8：裸命令可以接受，优先推断（配 act_first 效果最好）
- 0.9-1.0：任何命令都不需要追问

---

## 三、多意图指令（一句话多个动作）

**核心特征：** 老板一句话包含了完整的任务流。

| 模式 | 例子 |
|------|------|
| 然后连接 | "查一下状态然后把日志下载下来再整理成表格" |
| 同时连接 | "同时把 config 也更新了" |
| 中文逗号分割 | "查一下，整理好，发我" |
| and/then 英文 | "check the logs and format the output and send it to me" |

**校准变量：** `multi_intent_handling`
- parallel：同时做，汇报合并结果
- sequential：按顺序逐个做并汇报结果
- ask：先确认优先级再动手

---

## 四、粘贴式指令（先贴数据再说要求）

**核心特征：** 老板直接丢东西过来，默认你知道接住。

| 模式 | 例子 |
|------|------|
| 贴日志 | 一段 ERROR/WARN/INFO 日志 |
| 贴 JSON | API response / 配置文件 |
| 贴代码 | 代码片段（常附带"看看这个"） |
| 贴路径 | 文件路径（常附带"这里面是什么"） |
| 贴报错 | 报错信息（常附带"为什么"） |

**校准变量：** `paste_behavior`
- auto_analyze：自动分析粘贴内容，汇报结论
- ask_intent：先问"你想让我做什么"

---

## 五、简化/缩写式指令（中英文混缩）

**核心特征：** 老板用最短时间表达最多信息。

| 模式 | 例子 |
|------|------|
| 英文术语缩写 | "把 PR 提了"、"发 CR"、"check staging" |
| 拼音缩写 | "把 zt 改了"（状态）、"看下 xm"（项目） |
| 混合语序 | "这个 feature deploy 到 staging"、"server 的 config 改一下" |
| 数字标识 | "issue #7 看一下"、"PR #42 合并" |

---

## 六、状态/反馈型指令（没有明确动作）

**核心特征：** 老板不是在发指令，是在问你状态。

| 模式 | 例子 | AI 正确反应 |
|------|------|------------|
| 进度询问 | "怎么样了"、"到哪了"、"进度" | 汇报当前状态，不是问"你说什么" |
| 确认询问 | "好了吗"、"搞定没"、"done?" | 确认是否完成，不是执行 |
| 草草阅读 | "嗯"、"看到了"、"行" | 回复确认，等待下一步指令 |
| 半截话 | "那个……算了"、"先这样吧" | 停止当前工作，等待新的指令 |

## 附录：`precise_boss` vs `efficiency_boss` 风格对比

| 场景 | efficiency_boss | precise_boss |
|------|----------------|--------------|
| "修好它" | 直接修最近那个 Bug，附带一句"我修了那个 XXX，确认一下" | "你说的'它'是指上一个 issue 吗？确认后我马上改" |
| "优化" | 对最近讨论的代码做性能分析，汇报优化方案 | "你指的是哪个模块的优化？我列几个方向你选" |
| "贴一段 JSON" | 自动解析结构，汇报关键数据和潜在问题 | "我收到了 JSON，你想让我做什么——分析/转换/比对？" |
| "查一下顺便把日志下载了" | 两个都做，回来统一汇报 | "我先查状态，确认没问题再下载日志，你看顺序行吗？" |
